# 🌿 Branches e Merge

## O que é uma branch?
Uma branch é uma linha paralela de desenvolvimento. Permite trabalhar
em novas funcionalidades sem afetar o código principal (main).

## Comandos
- `git branch` — lista as branches existentes
- `git checkout -b nome-da-branch` — cria e muda para uma nova branch
- `git checkout main` — volta para a branch main
- `git merge nome-da-branch` — traz as mudanças da branch para a atual
- `git branch -d nome-da-branch` — apaga a branch (após o merge)

## Fluxo praticado neste repositório
1. Criei a branch `anotacoes-branches`
2. Adicionei este arquivo nela
3. Voltei para a `main` e fiz o merge
4. Apaguei a branch após a fusão

## 💡 Aprendizado prático
No VS Code, o nome da branch atual aparece no canto inferior esquerdo.
Sempre confira em qual branch você está antes de commitar!
